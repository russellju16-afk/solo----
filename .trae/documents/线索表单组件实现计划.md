# 线索表单组件实现计划

## 1. 创建目录结构

- 创建 `src/types/` 目录用于类型定义
- 创建 `src/services/` 目录用于API服务
- 创建 `src/components/forms/LeadForm/` 目录用于表单组件

## 2. 实现类型定义

在 `src/types/lead.ts` 中定义：
- ChannelType（客户渠道类型）
- CategoryInterest（意向品类）
- MonthlyVolumeSegment（月采购量区间）
- LeadPayload（线索提交数据结构）

## 3. 实现服务层

1. 在 `src/services/http.ts` 中封装 axios 实例
2. 在 `src/services/leads.ts` 中实现线索提交API

## 4. 实现通用 LeadForm 组件

在 `src/components/forms/LeadForm/index.tsx` 中实现：
- 基于 React Hook Form + AntD 的可复用表单
- 支持完整版和简化版（compact）模式
- 响应式设计，适配移动端和PC端
- 表单校验和提交处理
- 加载状态和错误处理

## 5. 集成到页面中

1. 首页（Home）底部添加简化版表单
2. 联系页（Contact）添加完整版表单
3. 产品详情页（Products/Detail）添加表单并传入产品ID

## 6. 确保代码质量

- 严格遵循 TypeScript 类型检查
- 确保组件可复用性和灵活性
- 实现良好的用户反馈机制
- 适配移动端和PC端布局

## 7. 测试验证

- 验证表单提交功能
- 验证不同页面的集成效果
- 验证响应式布局
- 验证表单验证和错误提示

## 技术栈

- React 18
- TypeScript
- React Hook Form 7
- Ant Design 5
- Axios
- Tailwind CSS

## 预期结果

- 完成通用线索表单组件的开发
- 实现线索提交功能
- 集成到多个页面
- 提供良好的用户体验
- 代码结构清晰，易于维护和扩展
