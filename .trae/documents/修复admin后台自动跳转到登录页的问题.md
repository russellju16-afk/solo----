## 修复admin后台自动跳转到登录页的问题

### 1. 修改 http.ts 的 401 处理逻辑

**问题**：http.ts 中的响应拦截器在检测到 401 状态码时，会自动调用 logout() 并跳转到登录页，导致切换菜单时页面闪退。

**修复方案**：
- 在 401 分支中保留错误提示
- 不再调用 logout()
- 不再调用 window.location.href = '/login' 或 navigate('/login')
- 仅仅 return Promise.reject(error)
- 增加控制台日志便于排查

**修改文件**：`admin/src/services/http.ts`

### 2. 确保 MainLayout 的登录检查逻辑正确

**现状**：MainLayout.tsx 中的登录检查逻辑已经符合要求，只在 store 中没有 token 且 localStorage 也为空时才跳转登录页。

**确认内容**：
- 登录检查 useEffect 逻辑正确
- 退出登录按钮处理正常

### 3. 检查其他可能的跳转登录逻辑

**现状**：通过全局搜索，只找到了登录页和退出登录按钮两处主动跳登录的场景，符合要求。

**确认内容**：
- 登录页本身的跳转逻辑正常
- 退出登录按钮的跳转逻辑正常
- 没有其他地方隐式调用 logout 或跳转 /login

### 4. 验证修复效果

**预期效果**：
- 清空浏览器 localStorage 中项目相关项，重新打开 admin，访问任何后台路由都会被重定向到 /login
- 登录成功后进入仪表盘，连续切换多个菜单不会莫名其妙被弹回登录页
- 如果某个接口因为权限配置问题返回 401，页面顶部会出现统一的错误提示，控制台会出现 [ADMIN HTTP 401] 日志，但当前登录态不会立即被清掉，不会自动跳转到登录页
- 只有在主动点击“退出登录”或清空 localStorage 中的 token 时，才会再回到登录页
