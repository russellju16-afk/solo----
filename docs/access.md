# 访问前台与后台接口

此前台默认入口 `http://localhost:5173` 和后台接口 `http://localhost:3002/api` 需要对应服务运行后才能访问。如果直接打开链接无法响应，通常是开发服务尚未启动或端口被占用。

## 启动前台（Vite）
1. 安装依赖：在仓库根目录执行 `npm install`。
2. 启动开发服务：运行 `npm run dev`（如需外网访问可加 `--host 0.0.0.0 --port 5173`）。
3. 浏览器访问：默认地址为 `http://localhost:5173`。

## 启动后台 API（NestJS）
1. 进入后端目录：`cd server`。
2. 安装依赖：`npm install`。
3. 启动开发服务：`npm run start:dev`（默认监听 `http://localhost:3002`，接口前缀 `/api`）。
4. 接口示例：健康检查 `http://localhost:3002/api` 或实际业务接口路径。

## 常见原因与排查
- **服务未启动**：确认上述命令已执行，并在终端中看到 Vite/NestJS 的启动日志。
- **端口占用**：若 5173 或 3002 已被占用，可在命令中指定其他端口并对应更新访问地址。
- **网络/代理限制**：容器或远程环境中可能存在端口转发要求，需确保本地能够访问到对应端口。
